🚨 현재 에러:
   sh: line 1: cd: packages/web: No such file or directory

✅ 해결 방법:
   Vercel Settings에서 Root Directory를 변경하세요!

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   URL: https://vercel.com/johns-projects-bf5e60f3/sajuapp-web/settings/general

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📍 2단계: Build & Development Settings 찾기

   페이지를 아래로 스크롤하여 다음 섹션 찾기:
   "Build & Development Settings"

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📍 3단계: OVERRIDE 버튼 클릭

   Framework Preset 옆에 있는 "Override" 버튼 클릭

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📍 4단계: 다음 값 정확히 입력

┌─────────────────────────────────────────────────────────┐
│ Framework Preset:                                       │
│   Vite                                                  │
├─────────────────────────────────────────────────────────┤
│ Root Directory:                                         │
│   packages/web                                          │
├─────────────────────────────────────────────────────────┤
│ Build Command:                                          │
│   npm install && npm run build                          │
├─────────────────────────────────────────────────────────┤
│ Output Directory:                                       │
│   dist                                                  │
├─────────────────────────────────────────────────────────┤

│   npm install                                           │
└─────────────────────────────────────────────────────────┘

⚠️  중요: 정확히 위와 같이 입력하세요!

📍 5단계: 저장

   "Save" 버튼 클릭 → 녹색 체크마크 확인

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📍 6단계: 재배포

   1. 상단 탭에서 "Deployments" 클릭
   2. "Redeploy" 버튼 클릭
   3. "Use existing Build Cache" 체크 해제
   4. "Redeploy" 확인 버튼 클릭

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📍 7단계: 빌드 로그 확인

   배포 시작됨 → 빌드 로그 실시간 확인

   ✅ 성공 시:
      "Build Completed" 메시지 표시
      Status: Ready

   ❌ 실패 시:
      에러 메시지 확인 후 알려주세요

💡 왜 이렇게 하나요?

   Root Directory를 "packages/web"로 설정하면:
   - Vercel이 packages/web 폴더를 프로젝트 루트로 인식
   - "cd packages/web" 명령 불필요
   - 더 단순하고 안정적인 빌드
